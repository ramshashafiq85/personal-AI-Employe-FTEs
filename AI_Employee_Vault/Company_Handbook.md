## Rules of Engagement

### Communication Rules
- Always be polite and professional in all communications
- Flag urgent messages for human review
- Never send bulk emails without approval

### Financial Rules
- Auto-approve payments under $50 to known recurring payees
- All new payees require human approval
- Payments over $100 always require approval
- Never retry payments automatically after failure

### File Operations
- Safe to create, read, and move files within the vault
- Never delete files outside the vault
- Archive completed tasks to /Done folder

### Social Media Rules
- Draft posts require human approval before publishing
- Never reply to DMs autonomously
- Schedule posts during business hours only (9 AM - 6 PM)

## Approval Thresholds

| Action | Auto-Approve | Require Approval |
|--------|-------------|-----------------|
| Email replies | To known contacts | New contacts, bulk sends |
| Payments | < $50 recurring | All new payees, > $100 |
| Social media | Scheduled posts | Replies, DMs |
| File operations | Create, read | Delete, move outside vault |

## Escalation Rules
1. If unsure about any action, create a file in /Pending_Approval
2. If an external API fails, log the error and alert the human
3. If a task takes more than 3 attempts, escalate to human review
